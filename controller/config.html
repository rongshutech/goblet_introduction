参数配置

goblet 目前支持 INI、XML、JSON、YAML 格式的配置文件解析，但是默认采用了 INI 格式解析，用户可以通过简单的配置就可以获得很大的灵活性。

默认配置解析

goblet 默认会解析当前应用下的 bin/AppName.conf 文件。

通过这个文件你可以初始化很多 goblet 的默认参数,在使用 ini 类型的配置文件中, key 支持 section::key 模式：
[basic]
port = 9090
env ="dev"
read_timeout = 30
write_timeout = 30
public_dir = "public"
uploads_dir= "./uploads"
ignore_url_case=true
secret= "cX8Os0wfB6uxxxxxxxxxxxxxxxxxxcE9"
db_engine= "mysql"
version="0.0.1"
https=false
https_certfile=""
https_key=""

[cache]
enable=false
amout=1000

[log]
file=""

上面这些参数会替换 goblet 的基础参数，当然你也可以在配置文件中配置应用需要用的一些配置信息，例如下面所示的数据库信息：
[mysql]
host=""
user=""
password=""
name=""
port=""
connect_timeout=""
ka_interval=""

当使用数据库时，就应配置相应的数据库的参数，goblet支持的参数配置只有上面的8种，其中的"mysql"，对应的是basic中的db_engine，即如果使用的是orcal，则应将上述的"mysql"，换成[oci8]

不同级别的配置

在配置文件里面支持 section，可以有不同的 Runmode 的配置，默认优先读取 runmode 下的配置信息，例如下面的配置文件：

[basic]
port = 9090
env ="dev"

[dev]
port = 8080
[prod]
port = 8088
[test]
port = 8888
上面的配置文件就是在不同的 runmode 下解析不同的配置，例如在 dev 模式下，httpport 是 8080，在 prod 模式下是 8088，在 test 模式下是 8888。其他配置文件同理。解析的时候优先解析 basic.env 下的配置。

basic 配置

  • env

    应用的运行模式，可选值为 prod, dev 或者 test. 默认是 dev, 为开发模式，在开发模式下出错会提示友好的出错页面，如前面错误描述中所述。
  • port

    应用监听端口，默认为 8080。
  • https

    是否启用 HTTPS，默认是 false 关闭。当需要启用时，先设置 https = true，并设置 https_certfile 和 https_key
  • https_certfile

    开启 HTTPS 后，ssl 证书路径，默认为空。
  • https_key

    开启 HTTPS 之后，SSL 证书 keyfile 的路径,默认为空。
  • db_engine

    数据库引擎设置
  • version

    app版本设置

cache 配置

  • enable

    缓存设置，默认为false
  • amout

    缓存数量设置，默认为1000

log配置

  • file

    log文件位置，默认为空

db_engine

  • host

    数据库地址
  • port

    数据库的端口号
  • user

    数据库连接的用户名
  • password

    数据库连接的用户名相应的密码
  • name

    将要连接的数据库名
  • connect_timeout

    数据库连接超时设置
